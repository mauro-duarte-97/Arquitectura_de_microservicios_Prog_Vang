📋 Responsabilidades por contenedor:
  • frontend: Captura código, gestiona JWT, renderiza resultados clasificados.
  • backend-java: Autentica (JWT), orquesta el flujo, persiste en BD, llama al servicio Python.
  • postgres-db: Almacena usuarios y auditorías. Solo accesible desde backend-java.
  • inference-py: Recibe código, construye prompts, invoca al LLM, estructura el JSON de salida.
  • ollama-ai: Motor LLM local. Solo accesible desde inference-py.

❌ E1 - JWT inválido
Backend Java rechaza la request en el filtro de seguridad.
Respuesta: 401 + redirección a login.
❌ E2 - Fallo de persistencia
BD no disponible o violación de constraint.
Respuesta: 500 + mensaje genérico al usuario.
❌ E3 - Timeout Python
Microservicio de inferencia no responde a tiempo.
Respuesta: 504 + reintento (3 veces) + mensaje de demora.
❌ E4 - Error motor IA
Ollama caído o el modelo falla.
Respuesta: 502 + auditoría marcada FAILED en BD.

📋 Leyenda de Componentes

Frontend: Interfaz de Usuario
Backend Java: Spring Boot - Orquestador
IA Python: FastAPI - Inferencia LLM
Base de Datos: PostgreSQL/MySQL
Usuario: Actor externo
Error: Casos excepcionales

🔄 Tipos de Comunicación

━━▶ Comunicación síncrona HTTP/REST (JSON)
🔐 JWT: Autenticación por token en cada request
📥 Datos: Código fuente, JSON estructurado
⏱️ Timeout: 30s para llamadas al servicio de IA
