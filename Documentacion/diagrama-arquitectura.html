Estrategia de Tolerancia a Fallos y Manejo de Excepciones: Para evitar interrupciones y garantizar robustez, el sistema cuenta con filtros de seguridad y manejadores de excepciones globales que capturan errores internos, timeouts y caídas del modelo de IA, devolviendo siempre códigos HTTP semánticos y mensajes claros al desarrollador.
E1 - Auth

JWT Inválido / Expirado

El cliente envía una request con token ausente, corrupto o que ya superó los 60 minutos de vigencia.

Manejador: Spring Security Filter
Respuesta: 401 Unauthorized
E2 - Persist

Caída de Base de Datos

El contenedor de base de datos Postgres está saturado, desconectado o se incumple alguna constraint de datos.

Manejador: GlobalExceptionHandler
Respuesta: 500 Internal Server Error
E3 - Network

Timeout en Python / FastAPI

El backend de inferencia Python demora más de 30 segundos en procesar el código o se encuentra inaccesible.

Manejador: RestTemplate Interceptor
Respuesta: 504 Gateway Timeout
E4 - AI Engine

Modelo de IA no responde

El motor de Ollama local está inicializándose o colapsó debido a falta de memoria RAM/VRAM en el host.

Manejador: FastAPI HTTP Exception
Respuesta: 502 Bad Gateway
